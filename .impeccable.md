# RLI — Design Context

## Target audience
Technical power users — solo developers and small-team engineers who already live in CLI coding agents (Claude Code, Codex, Gemini CLI, aider). They run multiple agent sessions in parallel, ship fast, prefer keyboard over mouse, are comfortable in a terminal, and have strong taste in tooling. Reference users: people who chose Warp over iTerm, Cursor over VS Code, Linear over Jira, Superhuman over Gmail.

They are *not* beginners. The product should not explain itself. It should reward fluency.

## Use cases
- Spawn multiple `claude` / `codex` instances inside one project, each in its own git worktree on its own branch, work on parallel tracks without collisions.
- Switch between projects (folders/repos) on the left rail; switch between agent sessions via tabs at the top.
- Watch one agent work in the center pane while reading the file it's editing in the right pane while running `bun run dev` in another pane.
- Highlight a piece of code → ⌘L → ask a small embedded model "why is this here?" → get an inline answer in the margin.
- ⌘⏎ to commit with an AI-generated message you can edit and approve.
- ⌘⇧B to pop a browser pane (GStack-driven) when running e2e tests.
- ⌘⇧; to see every connected skill and MCP across `~/.claude/`.
- Switch tabs and instantly see a one-line summary of what that session was last doing.

The constant verb is **swap context fast without losing your place**. Speed and recall are the product.

## Brand personality / tone

The intersection of three references:
- **Cursor** — workspace IDE feel, not just a terminal.
- **Warp** — terminal as a first-class native surface, not a forgotten tab.
- **Superhuman** — speed-fetishist, keyboard-first, dark elegance, almost no visual noise, every pixel earns its place.

If you had to put it in one sentence: **a confident dark instrument**. Quiet, not flashy. Dense with information but never busy. Type-led, not chrome-led. Animations are punctuation, not decoration.

What it must NOT feel like:
- Cyan/purple "AI app" aesthetic. No glow. No gradient borders. No glass.
- VS Code's homely activity bar with tooltips and status bars everywhere.
- Generic shadcn neutrals.
- A web app pretending to be a desktop app.

What it must feel like:
- A precision instrument that loads instantly, responds before you finish thinking, and gets out of your way.
- "Boring on purpose" surfaces (terminal, editor) framed by chrome that has *quiet personality* — a single distinctive serif/grotesque pair, deliberate spacing rhythm, monochrome with one cold accent reserved for active/focused state, not splashed everywhere.
- Like Linear in restraint, like Warp in surface quality, like Superhuman in keyboard density.

## Hard design constraints (from spec + conversation)

- **Single dark theme.** No light mode v1. No theme switcher.
- **macOS only v1.** Native macOS metrics (44pt header bar feel, no Windows-style chrome).
- **Animations on chrome only** — rails, tabs, palette mount, modal entry, hover/focus, dock transitions. **Never** on terminal contents (`xterm.js`) or editor contents (CodeMirror 6). Those surfaces are inert by design.
- **No telemetry, no analytics.**
- **Keyboard-first.** Every command has a chord, listed in palette. Mouse is fallback.
- **No emoji** anywhere in the chrome (icons only).
- **No light mode toggle, theme picker, account UI, or notification center.**

## Differentiation — the one unforgettable thing

The session tabs. Each tab shows the session's name (auto-generated from your first prompt to the agent), and underneath it in 11px is a single-line summary of what the agent is currently doing — updated every time the agent goes idle. You can glance at the tab strip and read what every parallel session is working on. No other tool does this. It should look effortless and feel slightly magical.

## What the system should optimize for

In order:
1. **Time-to-first-byte on every interaction.** No spinners. No "loading…" states for chrome.
2. **Information density without visual fatigue.** You should be able to look at RLI for 8 hours.
3. **Muscle memory.** Layouts and shortcuts should be predictable enough to use without looking.
4. **Beauty as a side effect of restraint**, not as a layer of polish on top.
